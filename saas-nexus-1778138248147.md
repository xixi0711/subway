我给你一个现实可落地、适合本科毕设的本地部署方案，不会吹过头，也不会让你踩大坑。

一、先把系统拆清楚（核心架构）
你的系统本质是：
代码用户问题（自然语言）        ↓大模型理解（NLU）        ↓判断：查数据库？查知识库？还是直接回答？        ↓RAG / SQL / 推理        ↓生成自然语言答案
👉 标准结构：
代码前端（Web/聊天界面）        ↓后端（FastAPI / Flask）        ↓调度层（Agent / Router）   ├── 大模型（LLM）   ├── 向量数据库（RAG）   ├── 结构化数据库（SQL）        ↓最终回答生成

二、本地大模型推荐（重点）
你这个项目不需要追求最强模型，而是要：

✅ 能跑
✅ 能调用工具（SQL/RAG）
✅ 中文能力可以
✅ 推理稳定

三、本地部署方式（非常关键）
⭐ 最简单方案（强烈推荐）
👉 Ollama + LangChain
1️⃣ 安装 Ollama
Bashcurl -fsSL https://ollama.com/install.sh | sh
2️⃣ 拉模型
Bashollama pull qwen2.5:7b

3️⃣ Python调用
Pythonfrom langchain_community.llms import Ollamallm = Ollama(model="qwen2.5:7b")response = llm.invoke("轨道交通信号系统是什么？")print(response)

⭐ 进阶方案（更专业）

Bashpip install vllm

四、RAG知识库方案
你这个是重点（毕设核心亮点）

⭐ 推荐技术栈
模块推荐向量模型bge-small-zh / bge-base向量库FAISS（本地）框架LangChain / LlamaIndex

⭐ 流程
代码文档（轨道交通资料）        ↓分块（chunk）        ↓Embedding（向量化）        ↓存入 FAISS

⭐ 查询流程
代码用户问题   ↓向量检索（TopK）   ↓拼接上下文   ↓喂给 LLM

⭐ 示例代码（简化版）
Pythonfrom langchain.vectorstores import FAISSfrom langchain.embeddings import HuggingFaceEmbeddingsembedding = HuggingFaceEmbeddings(model_name="BAAI/bge-small-zh")db = FAISS.load_local("faiss_db", embedding)docs = db.similarity_search("地铁信号系统", k=3)

五、数据库查询（SQL + LLM）
这是你“智能问答”的第二大亮点。

⭐ 推荐方案
👉 LLM → 自动生成 SQL → 查询数据库

⭐ 示例
用户：

北京地铁1号线长度是多少？

流程：
代码LLM → 生成SQL：SELECT length FROM metro_line WHERE name='1号线'数据库返回 → 42kmLLM → 转成自然语言

⭐ 工具
👉 LangChain SQL Agent
Pythonfrom langchain.agents import create_sql_agent

六、核心能力：Agent（最重要）
你系统的“智能”来自这里：

⭐ 三种能力

⭐ 推荐实现
