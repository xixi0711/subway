# 地铁智能问答系统毕设进展报告

## 第一周：项目基础架构搭建与核心功能实现

### 完成内容

1. **项目初始化**
   - 搭建FastAPI + SQLite基础架构
   - 配置Python虚拟环境和依赖管理
   - 创建项目目录结构

2. **数据库设计**
   - 设计并创建9个核心数据表：线路信息表、站点信息表、列车时刻表、交路信息表、乘客刷卡数据表、换乘关系表、站点地理信息表、线路路径信息表、客流统计表
   - 建立表间关联关系，确保数据完整性

3. **后端API开发**
   - 实现线路管理CRUD接口（`/api/lines`）
   - 实现站点管理CRUD接口（`/api/stations`）
   - 实现时刻表管理CRUD接口（`/api/timetables`）
   - 实现刷卡记录管理CRUD接口（`/api/card-records`）
   - 实现线网数据接口（`/api/network/*`）
   - 实现客流统计接口（`/api/passenger-flow/*`）

### 技术要点

- 采用FastAPI框架，实现高性能异步API
- 使用SQLite轻量级数据库，简化部署
- 实现RESTful API设计，标准化接口格式

## 第二周：AI智能问答功能与RAG增强

### 完成内容

1. **AI集成**
   - 集成火山引擎豆包API
   - 实现基础AI问答接口（`/api/ai-query`）
   - 构建地铁领域知识上下文

2. **RAG知识库模块**
   - 设计并实现RAG架构
   - 集成LangChain + Chroma向量数据库
   - 支持TXT文本文档解析
   - 实现启动时自动构建知识库

3. **缓存优化**
   - 实现LRU缓存机制
   - 优化数据库知识缓存
   - 实现回答缓存，提升响应速度

4. **意图识别**
   - 实现基于规则的意图分类
   - 区分结构化查询、非结构化知识查询、闲聊三类意图
   - 构建快速路径机制，减少API调用

### 技术要点

- 首次引入RAG技术，实现文档检索增强
- 多级缓存策略，响应时间从3-5秒降至毫秒级
- 意图识别路由，智能分配处理路径
- 降级机制：Chroma不可用时自动降级为关键词匹配

## 第四周：本地LLM集成与实时查询优化

### 完成内容

1. **本地LLM集成**
   - 集成Qwen2.5-1.5B本地模型

   - 支持本地/外部API双模式切换

2. **GPU加速配置**
   - 配置CUDA支持，优化GPU推理性能
   - 降级Python版本至3.11.9以兼容CUDA
   - 实现GPU/CPU自动切换机制
   - 测试GPU推理效果

3. **实时查询优化**
   - 禁用回答缓存，确保数据实时性
   - 优化数据库知识上下文构建
   - 增强提示词设计，提升模型理解能力
   - 实现回答格式清理，移除多余标签

4. **前端功能增强**
   - 新增清除缓存按钮
   - 优化AI问答界面显示
   - 提升用户体验

5. **模型优化与测试**
   - 优化生成长度，减少推理时间
   - 禁用采样，提升回答准确性
   - 添加回答后处理逻辑
   - 测试不同问题类型的回答效果

### 技术要点

- 本地LLM集成，实现离线推理
- 4bit量化技术，优化内存占用
- GPU加速，提升推理速度
- 实时查询机制，禁用缓存确保数据新鲜度
- 双模式设计，支持本地模型/外部API切换
