# 地铁智能问答系统技术栈文档

## 1. 技术栈概览

### 1.1 后端技术
- **编程语言**：Python 3.10+
- **Web框架**：FastAPI 0.135.3
- **服务器**：Uvicorn
- **数据库**：SQLite 3
- **API交互**：RESTful API
- **AI集成**：火山引擎豆包 API
- **缓存**：LRU内存缓存

### 1.2 RAG增强技术
- **Embedding模型**：BAAI/bge-large-zh-v1.5 (HuggingFace)
- **向量数据库**：ChromaDB（可选，降级使用关键词匹配）
- **RAG框架**：LangChain + LangChain-Community
- **文档解析**：TXT文本

### 1.3 前端技术（现代化）
- **框架**：Vue 3.4 (Composition API)
- **构建工具**：Vite 5.0
- **UI组件库**：Ant Design Vue 4.0
- **路由**：Vue Router 4
- **状态管理**：Pinia 2.1
- **HTTP客户端**：Axios
- **CSS预处理器**：Sass

### 1.4 开发工具
- **包管理**：pip + npm
- **环境管理**：Python虚拟环境 (venv)
- **代码版本控制**：Git

## 3. 技术架构

### 3.1 系统架构
- **前后端分离**：Vue 3前端 + FastAPI后端API
- **数据流向**：前端 → API → 数据库/知识库 → API → 前端
- **AI集成**：后端调用火山引擎API进行智能问答
- **RAG增强**：结合向量数据库检索与非结构化文档
- **前端代理**：Vite开发服务器代理 `/api` 请求到后端

## 4. 技术特点

### 4.1 后端特点
- **FastAPI框架**：高性能、自动生成API文档
- **SQLite数据库**：轻量级、无需额外配置
- **RESTful API设计**：标准化接口
- **CORS支持**：允许前端跨域请求
- **多级缓存**：回答缓存 + 数据库知识缓存
- **意图识别**：自动分类问题类型（结构化/非结构化/闲聊）
- **模块化设计**：缓存、RAG、意图识别解耦

### 4.2 RAG增强特点
- **语义检索**：基于向量相似度匹配（ChromaDB）
- **降级机制**：Chroma未安装时自动降级为关键词匹配
- **增量构建**：支持追加新文档
- **启动自构建**：后端启动时自动构建知识库

### 4.3 前端特点（现代化）
- **Vue 3 Composition API**：更灵活的组件逻辑
- **Vite构建工具**：极速开发体验，热更新
- **Ant Design Vue**：专业级UI组件库
- **Pinia状态管理**：现代化的状态管理
- **Vue Router**：单页面应用路由
- **响应式设计**：适配不同设备
- **Axios封装**：统一的API调用管理

### 4.4 AI集成特点
- **语义识别**：基于火山引擎豆包API
- **领域知识**：结合数据库中的地铁信息
- **RAG增强**：结合知识库检索提升回答质量
- **智能问答**：理解用户意图并提供准确回答

## 7. 技术风险与应对

### 7.1 技术风险
- **AI API依赖**：依赖火山引擎API的可用性
- **Embedding模型**：首次加载需下载约1GB模型
- **SQLite性能**：在大规模数据下可能存在性能瓶颈
- **向量检索精度**：依赖文档质量和分割策略

### 7.2 应对措施
- **API容错**：添加API调用失败的fallback机制
- **缓存策略**：多级缓存减少API调用次数
- **意图识别**：简单问题走快速路径，避免不必要的API调用
- **降级机制**：Chroma不可用时自动降级为关键词匹配

## 8. 技术演进建议

### 8.1 后端演进
- **数据库升级**：考虑使用PostgreSQL等更强大的数据库
- **缓存升级**：考虑使用Redis缓存以支持分布式部署
- **认证系统**：添加用户认证和权限管理

### 8.2 RAG能力演进
- **本地模型**：考虑使用本地部署的Embedding模型
- **知识图谱**：结合Neo4j构建地铁知识图谱
- **多模态**：支持图片、表格等非文本内容检索

### 8.3 前端演进
- **状态管理**：使用Pinia管理复杂状态
- **组件库**：使用Ant Design Vue组件提高开发效率
- **构建优化**：使用Vite提高开发体验

### 8.4 AI能力演进
- **多轮对话**：支持对话上下文和历史记忆
- **个性化**：根据用户历史查询提供个性化回答
- **本地模型**：考虑使用Ollama等本地LLM减少API依赖

**后端**：FastAPI + SQLite + RAG知识库
- 高性能API服务
- 多级缓存优化
- 意图识别智能路由
- RAG增强问答

**前端**：Vue 3 + Vite + Ant Design Vue + Pinia
- 现代化开发体验
- 专业级UI组件
- 清晰的状态管理
- 组件化架构
